# 设置页面模型启动参数自动保存

## 目标
移除"保存设置"按钮，修改参数后自动保存生效。

## 改动范围
仅修改 `src/settings.html` 一个文件。

## 改动内容

### 1. 移除保存按钮（line 799）
删除 `<button class="btn-save" onclick="saveParams()">保存设置</button>` 这一行。

### 2. 添加自动保存事件监听
在 `<script>` 末尾（`init()` 调用之前或之后），添加一个 `change` 事件监听器：

```js
// 模型启动参数面板内所有输入变化自动保存
document.getElementById('panel-launch-params').addEventListener('change', function(e) {
  const tag = e.target.tagName;
  if (tag === 'INPUT' || tag === 'SELECT' || tag === 'TEXTAREA') {
    // 排除自定义 GPU 层数输入框的 change 事件（其联动逻辑已由 n_gpu_layers 的 change 触发）
    if (e.target.id === 'n_gpu_layers_custom') return;
    saveParams();
  }
});
```

这样任何 input/select/checkbox 的 `change` 事件都会触发保存。

### 3. 保留恢复默认按钮
"恢复默认"按钮（line 800）保留，在 `resetParams()` 末尾添加自动保存调用，使恢复默认后配置也能立即生效。

### 4. 文案调整（可选）
- `onPresetModeChange()` 中的 `desc` 文本（line 454）："选择后自动填充并保存采样参数，可手动微调后点保存" → "选择后自动填充并保存采样参数，可手动微调后自动保存"
- `saveParams()` 保存成功的 toast 提示（line 1091）："设置已保存，重启模型后生效"（保持不变，仍适用于自动保存场景）

## 关键细节
- 使用事件委托（`change` 事件冒泡）而非每个元素单独绑定，简洁且维护成本低
- `change` 事件对于 `<input type="number">` 在失去焦点且值改变时触发，对于 `<select>` 和 `<input type="checkbox">` 立即触发，符合"修改后自动保存"的需求
- 对于需要实时响应的输入（如 `onPresetModeChange`），已直接调用 `saveParams()`

## 验证方式
1. 直接打开 `src/settings.html` 确认保存按钮已消失
2. 修改任意参数（如温度、线程数、下拉选择等），确认保存 toast 提示出现
3. 切换推荐模式，确认自动保存仍然工作
4. 点击"恢复默认"，确认自动保存被调用
