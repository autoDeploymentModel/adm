# 2026-07-18

## 修复：agent 页鼠标滚轮失效（最终结论）

### 排查链路
1. 原始代码在 agent.html 用捕获阶段 `wheel` 监听 `preventDefault+stopPropagation` 掐断滚轮 → TUI 收不到滚动。
2. 移除拦截器后仍不生效：xterm 仅在鼠标追踪协议带 wheel 位时才转发；admAgent 二进制确认用 `?1006h(SGR)+?1003h`，格式应匹配。
3. 加 window 捕获转发器 + SGR 报文 → 仍不滚。加临时调试浮标确认：**代码在跑、事件被捕获、数据经 `agent_terminal_input` 写入 PTY**（浮标可见）。
4. 关键验证：把滚轮改成发**方向键** `▲=\x1b[A / ▼=\x1b[B` → TUI **正常滚动**。说明 PTY 输入通道 100% 通，问题就是 **SGR 鼠标报文不被该 TUI 的滚动逻辑接受**。

### 最终方案（已落地 `src/agent.html`）
- 滚轮 → 翻译成方向键序列（`\x1b[A` 上 / `\x1b[B` 下）经 `agent_terminal_input` 写入 PTY。
- 在 `window` 捕获阶段拦截 `wheel`，`preventDefault + stopPropagation`（仅 wheel，不影响点击/拖选）。
- deltaY 统一换算像素、跨 ~24px 阈值发一次，兼容行/页 deltaMode、避免触控板刷屏。
- 已删除调试浮标与诊断注释，代码清理为正式版。

### 关键经验（务必记住）
- **agent iframe 只加载一次**（`index.html` 的 `agentFrameLoaded` 守卫，`src` 不重设）。改 `src/agent.html` 后必须**重启 `pnpm tauri dev` / 整个应用**才能让 iframe 重新加载新代码，否则反复进 Agent 页仍是旧 JS —— 这正是前几次「改了不生效」的主因。
- 后端 `agent_terminal_input` 把 base64 解码后 `write_all` 到 PTY master，与键盘同链路；键盘能打字即证明该通道通。
